# Tmux 環境設定

此儲存庫包含 `tmux` 設定檔和一個 `Makefile`，用於自動化設定流程。它能標準化您的 `tmux` 環境，確保 Tmux 插件管理器 (TPM) 和正確的設定都已到位。

---

## 📜 說明：`make tmux` 的功能

`make tmux` 指令會自動化整個設定流程。當您執行它時，它會執行以下操作：

1.  **安裝 Tmux 插件管理器 (TPM)**：它會檢查 `~/.tmux/plugins/tpm` 中是否已安裝 TPM。如果沒有，它會從官方 GitHub 儲存庫克隆下來。TPM 對於管理 `tmux` 插件至關重要。
2.  **連結設定檔**：它會從本儲存庫的 `tmux.conf` 創建一個符號連結到 `~/.tmux.conf`。這是 `tmux` 尋找其設定檔的標準位置。此方法允許您將設定檔在此目錄中進行版本控制，同時讓 `tmux` 可以存取它。

---

## 🚀 快速入門

### 前置要求

在開始之前，請確保您的系統已安裝以下軟體：
*   `git`
*   `tmux`

### 安裝

要設定您的 `tmux` 環境，請在終端機中導航到此目錄並執行：

```bash
make tmux
```

此腳本是冪等的，這意味著您可以安全地多次運行此指令。它只會在必要時執行操作。

### 如何管理插件 (透過 TPM)

此設定使用 **Tmux 插件管理器 (TPM)**。一旦您進入 `tmux` 會話中，可以使用以下快捷鍵來管理您的插件：

*   **安裝新插件**：按下 `Prefix + I` (大寫 'I')。
    *(這將獲取您在 `tmux.conf` 中添加到列表裡的任何新插件)*
*   **更新現有插件**：按下 `Prefix + U`。
    *(這將把所有已安裝的插件更新到最新版本)*

> **注意**：預設的 `Prefix` 是 `Ctrl + b`。

## ✨ Tmux 基本用法

以下是一些 `tmux` 的基本指令，幫助您快速上手。請記住，在每個指令之前都要按下 `Prefix` (`Ctrl + b`)。

| 快捷鍵             | 動作                       |
| ------------------- | -------------------------- |
| `Prefix` `c`        | 創建新視窗                 |
| `Prefix` `,`        | 重命名當前視窗             |
| `Prefix` `p`        | 切換到上一個視窗           |
| `Prefix` `n`        | 切換到下一個視窗           |
| `Prefix` `%`        | 垂直分割窗格               |
| `Prefix` `"`        | 水平分割窗格               |
| `Prefix` `d`        | 分離會話 (保留會話在背景執行) |
| `Prefix` `[`        | 進入複製模式 (用於捲動內容) |
| `tmux a`            | (在 Shell 中) 重新連接到上次會話 |
| `tmux new -s name`  | (在 Shell 中) 啟動一個新的命名會話 |
